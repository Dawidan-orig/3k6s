Блокировка, которая может быть захвачена только в одному потоке команд. Вообще, это [[Мониторы|монитор]].
Может быть только в одном потоке.
Используется, по сути, только в контексте прерывания.

Суть в том, что если другой процесс влезет в спин-блокировку, то он начнёт циклично проверять, когда ж ресурс наконец освободится.

Достаточно универсальное решение взаимоблокировок, но при неправильном использовании может быть долгим.

Для чтения и записи:
- При чтении нельзя, чтобы кто-либо записывал.
- При записи нельзя ещё дополнительно, чтоб кто-либо читал.
то-есть при записи полный lock.
