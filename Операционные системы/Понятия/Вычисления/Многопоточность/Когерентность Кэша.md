Разделение такое:
Есть вот ЦП, потом КЭШ L3 (Третий уровень) ( #Переписать  )
И пачка L2 с ядрами под ними.

(Редко Кэш может быть на регистры, я этого вообще не понял)

Вот делает линейную программу многопоточно.
Сначала используемая память копируется в L3, а уже оттуда - дальше в L2.

А потом линейные команды разбиваются.
a=a+1,
a=a\*2 в разных ядрах.

Потом вешаем ожидание через событие-узнавание.
Короче, логика когерентность кэша - это когда между всеми кэшами L2 есть коммуникация и синхронизация.

Как это достигается? -
[[Алгоритм MSI]]

С помощью специального ПО на ЦП, не ядер, для сохранения когерентности кэша.
