Это блокировка, которая указывает на свободные потоки (Или, более глобально-абстрактно, на свободные данные).
Она уведомляет процессы и способна переводить их в другие состояния:
Переводит процессы в состояние ожидания и будит следующий в очереди ожидания, когда освобождается.

Может работать сразу с большим количеством устройств/элементов памяти/чем угодно.

Пишут что-то вроде Semaphore = n, где n - это число контролируемых штук. Эта вещь -- счётчик использования. У [[Мютекс|мютексов]] он равен 1.

При попытке захвата захватывающий попадает в очередь ожидания и замораживается.
Используется для долгоожидающих штук. Ибо если время ожидания короткое - переводы из состояний в состояния будут превышать время выполнения.

Не обрабатываются планировщиком в [[Запрет Прерываний|контексте прерываний]], а потому используется только в контексте процесса

Операции:
down -- захват семафора (Через увеличение счётчика на 1)
up -- Освобождение.
sema_init -- создание в runtime
down_interruptible -- Попытка захвата семафора. При неудаче переходим в [[Состояния процесса|состояние]] INTERRUPTIBLE. Прерываемо - для семафора.
down_trylock -- Захват без блокировки.

Для чтения-записи только [[Мютекс]].
