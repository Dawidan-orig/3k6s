Это фиксированное количество нодов, которые имеют ссылки на GDS.
Используется в [[Кэши свободных типов данных]].

Один узел соответствует одному файлу, и содержит в себе разом все адреса блоков файла с адресом блока указателей. Причём каждый узел имеет пространство фиксированного количества дисковых адресов.
И файл может перерасти этот лимит, что потребует создания сложной адресно-файловой структуры (Вроде дерева)

Файлы на линуксе - это эти самый i-nod'ы. Они представляют собой ID файла.
Для них есть своя таблица, содержащая данные о них.

Сложность в сохранении пустых папок и создании каталогов:
Таковая в таблице будет просто i-node c именем.
Суть в том, что файлов не будет в каталоге, они хранятся где-то в более оптимальном месте.

Это лучше FAT тем, что в памяти в итоге держится только текущий файл.
Это лучше списка скоростью.
