Вопросы на 3.
Это система, на которой пишут модели. Инструмент моделирования.
Использует [[Модельное время]]

Он работает как стандартная вычислительная машина.

Работает по логике блоков. Блок - это {?Какое-то хранилище [[Транзакт||Транзакта]]?}

Пример программы:
`Simulate` - начало паботы
`Generate` {1 или другое число} - создание [[Транзакт||Транзакта]] (Со временем 1)
`Start` {1 или другое число} - забираем поток, начинаем выполнение.
`Terminate` {1 или другое число} - Удаляем [[Транзакт]]
`End`

terminate удаляет транзакт. Generate - создаёт со значением.
Как только значение `start`<=0, то основной поток освободиться.

Но если создать второй `start`, то его ничто не коснётся, так как он заблокирует только-только разблокированный основной поток.

Всего 8 параметров у каждой команды.
Например,
`start 1,,,1` даёт 1-й и 4-й параметр. Первый это значение ID для потоков, полагаю.
4-й отвечает за вывод состояния [[Основные Цепи, ЦТС-ЦБС]].
На лекции он вывел состояние [[Основные Цепи, ЦТС-ЦБС|ЦБС]], поскольку там находился транзакт.

Generate 1000 - создаёт транзакт с [[Модельное время]] = 1000.

Ещё пример:
```GPSS
Generate 1.5
Terminate

Generate 1000
Terminate 1

Start 1,,,1
```

В итоге вывело время 666, так как транзакты создавались каждые 1.5 ед. времени в течении 1000 ед. времени.

Изменяем
```GPSS
Generate 1.5
Advance 3 - Перемещаем транзакт в цепь будущих событий + значение 3
Terminate

Generate 2,,,1 - Один вывод для этого транзакта
Terminate 1

Start 1,,,1
```

Итого будут транзакты со временем: 1.5, 4.5 и ещё потом 2.

3) И ещё один 4.5, так как он влез в 3.5, это 1.5\*3(Generate)
