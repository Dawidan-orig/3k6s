Перемещаются по имитационной модели и указывают на блоки.
Он может запустить основной поток.

Тесно связано [[Основные Цепи, ЦТС-ЦБС|с основными цепями]]

Если указатель цепи указывает на пустое место, то это значит, что нет транзактов в системе.
А если их нет - можем двинуть [[Модельное время]] вперёд, используя время появления транзакта.
(У транзакта есть скрытый параметр, отвечающий за время его появления. Он устанавливается блоком Generate с помощью формулы {[[Модельное время]] + Значение в Generate})
Когда транзакт выходит из ЦБС в GENERATE - GENERATE всё-таки делает одно действие - помещает в ЦБС новый транзакт, с новым временем.

Когда в ЦТС не остаётся транзактов - из ЦБС забирается транзакт с минимальным указанным временем.

Если происходит ситуация ожидания критической секции - то прежде всего проверяем транзакты с минимальным временем появления (Из-за добавления в ЦТС по FIFO, именно они будут первыми - слева) (Сортировка тут вот такая, да)

Транзакт движется по модели. Что с ним может произойти?:
1) Он будет заблокирован (Команда Seize) и не будет передвигаться по модели
2) Перенос обратно в [[Основные Цепи, ЦТС-ЦБС|ЦБС]] (Advance)
3) Он будет удалён (Terminate)

создаётся в GENERATE ([[Контроль имитации, SIMULATE-START-TERMINATE-END|Контроль Имитации]]), параметры:
1) Частота
2) Смещение частоты (Случайное число)
3) Встроенный Advance (Отсрочка выхода первого транзакта)
4) Ограничение по количеству транзактов (Перестанет генерировать их, когда достигнется)
5) Приоритет
6) Количество изначально данных параметров в транзакте
7) Формат параметра (Ставим либо F для "Слова", либо H для "Полуслова" (Чойта такое - хз)), либо изначальное значение всех параметров (Возможно)
